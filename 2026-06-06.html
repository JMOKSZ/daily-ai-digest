01 本地 AI Agent 全面爆发：从 Gemma 4 到 TARS

本周最清晰的结构性信号来自 Google 的 Gemma 4 12B 和字节跳动的 TARS 桌面 Agent。前者以 Apache 2.0 开源了一个在 16GB 统一内存上即可运行的完整多模态模型——视觉直接嵌入 LLM 骨干，无需独立编码器，意味着用户可以在离线笔记本上获得与云端相当的理解能力。配合 QAT 量化版本（E2B 压缩至 1GB），端侧部署的门槛已从"实验性"降低到"产品级"。

而 TARS 的横空出世展示了更激进的可能性：一个完全本地运行、仅凭视觉截图操作桌面 GUI 的 Agent，29K Stars 验证了开发者社区对"隐私优先+自主操控"的强烈需求。它不依赖 DOM 解析或平台 API，意味着未来任何应用——无论是否提供 API——都可以被 Agent 操控。这标志着 Computer Use Agent 的范式从云端的"远程控制"转向本地的"无缝代理"。

深层信号：这两个项目合在一起定义了 2026 年 Q3 的竞争基线——多模态 + 本地化 + 离线运行 + 可操控一切。云 API 的"按 token 收费"模型面临挑战：当模型能在本地免费运行、无需网络、保护隐私，为什么还要为每次对话付费？预计未来三个月内，每个主流模型家族都会推出本地 Agent 版本。这将对 SaaS 商业模式和用户数据主权产生深远影响。

02 递归自我改进：AI 进化的"起飞点"临近

Anthropic 的 RSI 路线图可能是一个历史节点。其核心主张是：一旦 AI 系统能够可靠地帮助改进 AI 系统本身（包括架构设计、训练策略、数据筛选），进步速度将从"线性"变为"指数级"。Claude 目前已编写 Anthropic 超过 80% 的生产代码——这不只是辅助编程，而是 AI 深度参与自身基础设施的构建。RSI 的关键前提（AI 在其他系统上展示出接近人类的推理和改进能力）正在被多个实证数据支撑。

同时，Claude rsync 数据分析提供了重要佐证：在核心里程碑级的 C 代码库上，AI 辅助的提交不仅没有增加 bug，反而将 bug 率降至历史基线的 45% 左右（尽管统计显著性有限）。这与 Alibaba Open Code Review 的内部结论一致——经过两年、数百万缺陷的验证，AI 代码审查在工程化后实现了比纯人工审查更高的一致性。

RSI 的更大含义是经济层面的。当 AI 本身成为 AI 研发中最具生产力的"员工"，人才市场的价值将从"会编程的人"转向"能定义问题、设计奖励函数、构建评估框架的人"。同时，"超级对齐"问题的紧迫性急剧上升——一个自主改进的系统，如果在初始阶段有微小的偏向，可能在自循环中被指数级放大。Anthropic 与竞争实验室在生物武器防御上的合作，正是对这一风险的早期响应。

03 算力军备竞赛：$350 亿押注与芯片瓶颈

Anthropic 获得 350 亿美元债务融资（Apollo + Blackstone + Broadcom），专门用于租赁 TPU 算力——这是 AI 公司历史上最大规模的融资之一。结合 TSMC 明确表态"AI 芯片需求将连续多年超过供应"，两条新闻共同勾勒出当前 AI 行业的根本矛盾：模型能力仍在快速增长，但支撑其运行的物理资源正在成为稀缺瓶颈。

这种供给紧张正在重塑整个行业结构。Broadcom 作为 Anthropic 交易的后盾方，表明芯片设计公司正在从"卖芯片"转向"租算力"的金融化模式。这与此前 Microsoft 与 OpenAI 的算力租赁模式一脉相承。Cloudflare CEO 的"Bot + Agent 流量超越人类"则从另一个角度印证了这一趋势：计算不再是"人类使用软件"的成本，而是"AI 运行基础设施"的成本。

展望：当训练算力被锁定在长期金融合约中，小型创业公司和研究机构的进入壁垒会进一步抬高。但这也催生了另一条路径——本地模型优化（如 Gemma 4 QAT 带来的 10 倍压缩）和更高效的架构（MoE、蒸馏等）。未来 12-18 个月，AI 行业的竞争将从"谁的模型最大"转向"谁的单位算力产出最高"——无论是在云端租赁还是在本地运行。PG_durable 这样的基础设施工具，则是在应用层面最大化利用有限算力的微观案例。
